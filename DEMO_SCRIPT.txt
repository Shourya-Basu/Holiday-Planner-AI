[1:00 - ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's look at the system architecture. First, the user's input passes through a security checkpoint function node to verify safety before any LLM processing. If the request is safe, it routes to our orchestrator agent, which delegates specialized tasks to the budget agent and the itinerary agent using Agent Tools. These sub-agents query real-time data from our local Model Context Protocol server, which exposes tools like weather forecasting, attraction searches, currency rates, and packing lists. Finally, the orchestrator compiles a draft plan and halts at the human approval node, which implements a human-in-the-loop checkpoint before the final itinerary is formatted and generated.
